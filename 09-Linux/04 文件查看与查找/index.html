04 Linux 文件查看与查找

发表于2021-03-30|更新于2021-05-28|Linux
|字数总计:1.6k|阅读时长:6分钟|阅读量:

文件查看

  • cat：由第一行开始显示文件内容
  • tac：从最后一行开始显示，与 cat 相反
  • nl：显示的时候，同时显示行号
  • more：一页一页的显示文件内容
  • less：与 more 类似，比 more 更好的是，支持向前翻页
  • head：只看前面几行
  • tail：只看后面几行
  • od：以二进制的方式读取文件内容

cat

tac

将文件从最后一行到第一行反向在屏幕上显示出来。

nl：添加行号打印

more

一页一页翻动。

空格键：向下翻一页

Enter：向下翻一行

/字符串：代表在这个显示的内容中，向下查找字符串这个关键词

:f：立刻显示出文件名以及目前显示的行数

q：代表立刻离开 more，不再显示该文件内容

b 或 Ctrl+b：往回翻页，不过这个操作只对文件有用，对管道无用

less

一页一页翻动。

空格键：向下翻动一页

Pagedown：向下翻动一页

Pageup：向上翻动一页

/字符串：向下查找字符串的功能

?字符串：向上查找字符串的功能

n：重复前一个查找（与 / 或者 ? 有关）

N：反向的重复前一个从查找（与 / 或者 ? 有关）

g：前进到这个数据的第一行

G：前进到这个数据的最后一行去

q：离开 less

head

取出前面几行。

tail

取出后面几行。

od：非纯文本文件

执行文件通常都是二进制文件（Binary file），使用一般文件查看的命令来读取它的内容时，确实会产生类似乱码的数据。利用这个命令，可以将数据文件或是二进制文件的内容数据读出来。

文件的查找

which

这个命令是根据【PATH】这个环境变量所规范的路径，去查找执行文件的文件名。

whereis

locate / updatedb

locate 查找数据特别快，这是因为 locate 查找的数据是由已建立的数据库 /var/lib/mlocate/ 里面的数据所查找到的。不再需要去硬盘中读取数据。

这个数据库的建立默认是每天执行一次，每个 Linux 发行版都不同，但是CentOS 7.x 是每天更新数据库一次。所有当建立起来的文件，却还在数据库更新之前查找文件， locate 可能提示找不到。

  • updatedb：根据 /etc/updatedb.conf 的设置去查找硬盘内的文件，并更新 /var/lib/mlocate 内的数据库文件；
  • locate：依据 /var/lib/mlocate 内的数据库记录，找出用户所输入的关键词的文件名

updatedb 命令回去读取 /etc/updatedb.conf 这个配置文件的设置，然后去硬盘里进行查找文件名的操作，最后更新整个数据库文件。因为 updatedb 回去查找硬盘，所以这个命令可能会等待数分钟时间。

find

很强大，可配置参数很多，但是速度慢，可能会影响硬盘性能，所以一般不使用 find。

可配置的参数分三种：与时间有关参数、与使用者或用户组有关参数、与文件名或权限有关参数。

详细参数，暂略。
