代理模式概述

代理模式也叫委托模式，是一种使用率非常高的设计模式，其定义如下：

Provice a surrogate or placeholder for another object to control access to it.

为其他对象提供一种代理以控制对这个对象的访问。

代理模式中有三个重要的角色：

  • Subject 抽象主题角色

    可以说抽象类或者接口，是一个最普通的业务类型定义。

  • RealSubject 真实角色

    实现了抽象主题中的具体业务，是代理对象所代表的真实对象，是最终要引用的对象。就是它需要被代理。

  • Proxy 代理主题角色

    代理类，提供了与真实主题相同的接口，其内部含有对真实主题的引用，它可以访问、控制或扩展真实主题的功能。帮助真实角色处理业务逻辑完毕前后还可以做一些预处理或者善后的处理。

代理模式优点

职责清晰，可以使真是的角色专注于实现自己的业务逻辑，不用去关注一些公共的业务（日志、安全等）；

高扩展性，可以很方便的扩展真实主题的功能，如添加一些预处理或者善后的操作。

代理模式缺点

一个真实角色就会产生一个代理角色，，类的数量翻倍增加，代码量翻倍，开发效率变低；

增加了系统复杂度；

在调用端与真实主题之间增加了一层代理对象，可能会造成请求处理速度变慢。

动态代理

动态代理概述

根据代理的创建时期，代理模式分为静态代理和动态代理。

  • 静态：由程序员创建代理类或特定工具自动生成源代码再对其编译，在程序运行前代理类的 .class 文件就已经存在了；
  • 动态：在程序运行时，运用反射机制动态创建而成。

动态代理分为两大类：

  • 基于接口的动态代理——JDK 动态代理
  • 基于类的动态代理 —— cglib
  • Java 字节码实现 ——JavaAssist

需要了解两个类：

  • InvocationHandler：调用处理程序。接口里只有一个方法

  • Proxy：代理，生成、得到代理对象
